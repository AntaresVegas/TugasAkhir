\section{Hypertext Markup Language (HTML)}
\label{sec:html}
\subsection{Pengertian HTML}
 
\textit{Hypertext Markup Language} (HTML) merupakan bahasa markup yang digunakan untuk menyusun struktur dasar halaman web. HTML tidak termasuk bahasa pemrograman, melainkan bahasa yang digunakan untuk mendeskripsikan bagaimana konten ditampilkan pada browser. HTML memungkinkan pengembang untuk mengatur berbagai jenis konten seperti teks, gambar, tautan, tabel, dan elemen multimedia lainnya.

HTML berfungsi sebagai kerangka utama dari halaman web, yang menentukan bagaimana setiap bagian konten disusun sehingga dapat ditampilkan secara terstruktur oleh browser~\cite{duckett:11:htmlcss}.

HTML bekerja dengan konsep \textit{tag-based structure}, di mana setiap elemen ditandai dengan tag tertentu. Dengan adanya struktur ini, browser dapat memahami peran masing-masing elemen dan menampilkannya dengan benar.

\subsection{Elemen dan Tag HTML}
Elemen HTML merupakan komponen dasar dalam penyusunan halaman web. Setiap elemen biasanya terdiri dari tag pembuka dan tag penutup, serta isi di antara keduanya. Sebagai contoh, elemen paragraf dituliskan dengan <p> sebagai tag pembuka dan </p> sebagai tag penutup.

Selain itu, elemen HTML juga dapat memiliki atribut yang memberikan informasi tambahan. Atribut ini dituliskan di dalam tag pembuka, seperti atribut href pada elemen <a> untuk menentukan alamat tautan.

Struktur elemen yang saling bersarang ({\it nested elements}) memungkinkan terbentuknya hubungan hierarkis antar elemen. Hal ini menjadi dasar penting dalam pembentukan struktur DOM.

\subsection{Struktur dasar HTML}
\label{sec:struktur}
Dokumen HTML memiliki struktur standar yang terdiri dari beberapa bagian utama, yaitu <html>, <head>, dan <body>. Elemen <html> merupakan root dari seluruh dokumen. Elemen <head> berisi informasi metadata seperti judul halaman, sedangkan <body> berisi konten utama yang ditampilkan kepada pengguna.

Struktur ini memastikan bahwa setiap halaman web memiliki susunan yang konsisten dan dapat dipahami oleh browser.

\subsection{Hierarki elemen HTML}

Struktur HTML bersifat hierarkis, yang berarti elemen dapat berada di dalam elemen lain. Misalnya, elemen <div> dapat berisi elemen <p>, <img>, atau elemen lainnya. Hubungan ini membentuk struktur bertingkat yang menyerupai pohon.

Hierarki ini penting karena menentukan bagaimana elemen saling berhubungan dan bagaimana struktur tersebut akan diproses menjadi DOM. Semakin kompleks halaman web, maka semakin dalam dan luas struktur hierarki yang terbentuk.

\section{Document Object Model (DOM)}
\label{sec:dom}

 \subsection{Pengertian DOM}
\textit{Document Object Model }(DOM) adalah representasi dari dokumen HTML dalam bentuk struktur objek yang dapat diakses dan dimanipulasi oleh program. DOM memungkinkan setiap elemen dalam halaman web diperlakukan sebagai objek yang memiliki properti dan metode~\cite{marini:02:thedom}.

Menurut \textit{World Wide Web Consortium}, DOM menyediakan antarmuka standar untuk mengakses dan memodifikasi dokumen HTML.

Dengan adanya DOM, halaman web tidak hanya menjadi tampilan statis, tetapi juga dapat diproses dan dimodifikasi secara dinamis.

\subsection{Struktur DOM sebagai Tree}
Struktur DOM berbentuk pohon (tree structure), di mana setiap elemen HTML direpresentasikan sebagai node. Node paling atas disebut root node, yang biasanya adalah elemen <html>.
Setiap node dapat memiliki:
\begin{itemize}
    \item Parent (induk)
    \item Child (anak)
    \item Sibling (saudara) 
\end{itemize}
Struktur ini memungkinkan proses traversal dilakukan secara sistematis, seperti menggunakan DFS.

\subsection{Jenis node pada DOM}
DOM terdiri dari beberapa jenis node, antara lain:
\begin{itemize}
    \item Element node: merepresentasikan tag HTML
    \item Text node: merepresentasikan isi teks
    \item Attribute node: merepresentasikan atribut 
\end{itemize}
Keberadaan berbagai jenis node ini membuat struktur DOM lebih kompleks dibandingkan HTML mentah, sehingga analisis DOM memberikan informasi yang lebih lengkap.

\section{Struktur Data Graf}
\subsection{Pengertian Graf}
Graf adalah struktur data yang terdiri dari simpul (node) dan sisi (edge) yang menghubungkan simpul tersebut.

Graf digunakan untuk merepresentasikan hubungan antar objek dalam berbagai bidang komputasi~\cite{cormen:22:introalgo}.

\subsection{Graf Berarah}
Graf berarah adalah graf yang memiliki arah pada setiap edge. Dalam konteks DOM, hubungan parent ke child memiliki arah, sehingga dapat dimodelkan sebagai graf berarah.

\begin{figure} [H]
	\centering  
	\includegraphics[scale=0.65]{Skripsi/DokumenSkripsi/Gambar/digraph2.png}
	\caption[Contoh graf berarah sederhana (node dan edge)]{Contoh graf berarah sederhana (node dan edge)} 
	\label{fig:digraph} 
\end{figure}

\subsection{Representasi DOM sebagai graf}
Struktur DOM dapat direpresentasikan sebagai graf dengan cara memetakan setiap elemen HTML sebagai node dan hubungan parent-child sebagai edge.

Pendekatan ini memungkinkan struktur halaman web dianalisis menggunakan teori graf, serta mempermudah visualisasi menggunakan perangkat lunak seperti Graphviz.

\subsection{Pengertian Traversal Graf}
Traversal graf adalah proses mengunjungi setiap node dalam graf secara sistematis. Traversal diperlukan untuk memastikan seluruh struktur graf dapat diproses tanpa ada node yang terlewat.

\subsection{Depth-First Search}
{\it Depth-First Search} (DFS) adalah metode traversal yang bekerja dengan menelusuri node hingga ke bagian terdalam sebelum kembali ke node sebelumnya.

Metode ini sangat cocok untuk struktur berbentuk tree seperti DOM karena mengikuti pola parent-child secara mendalam.

Menurut {\it Introduction to Algorithms}, DFS sering digunakan untuk eksplorasi struktur hierarkis dan graf.
